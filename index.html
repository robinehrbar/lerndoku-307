Modul 307 - interaktive Webseiten

Einleitung

Wenn man die Grundlagen von HTML & CSS beherrscht, kann man sich an das, für Fortgeschrittene, PHP und JavaScript machen. Diese zwei Skriptsprachen bringen diverse Vorteile für die Interaktion mit Webseiten. Der wesentlichste Unterschied zwischen PHP und JavaScript ist, dass JavaScript auf dem Computer des Benutzers, mithilfe eines Browsers ausgeführt wird(clientseitig), wohingegen PHP auf einem Webserver läuft(serverseitig).

Arbeitsumgebung und Tools

Bevor man mit Skripten beginnt, sollte sichergestellt werden, dass man eine angenehme Arbeitsumgebung mit nützlichen Tools für sich vorbereitet. Essenzielle Tools wie diese dürfen nicht fehlen:

D.O.M.

Ist die Abkürzung für Document Object Model und es ist eine Methode, welche den Aufbau aller HTML-Dokumente als Baumstruktur darstellt. Jeder "Ast" stellt somit zum Beispiel einen Absatz oder einen Titel dar. Für Browser bildet dies einen wichtigen Baustein für dynamische Webseiten. Ganz gut sehen kann man es im Chrome Developer Tool. Mit Windows öffnet man dies mit [F12], bei Linux mit [Ctrl]+[Shift]+[J] und mit Apple geht es mit [⌘]+[Optionstaste]+[J].

PHP erklärt

PHP ist die Abkürzung für Personal Home Page Tools. Ihre Syntax ähnelt der von C und Perl, sie dient bekanntlich zur Erstellung von dynamischen Webseiten und vielen anderen Möglichkeiten wie zum Beispiel das Eintragen von Daten in Datenbanken, aber bei uns im Kurs ging es grösstenteils um Formulare. PHP ist nicht kostenpflichtig, sondern es kann gratis auf einem Webserver verwendet werden. Wie in der Einleitung schon erwähnt ist PHP serverbasiert, das heisst, dass alle Prozesse auf dem Webserver ablaufen und somit die Clients weniger effektive Rechenleistung auf ihrem Gerät vorweisen müssen.
Ursprünglich war PHP nur eine Ansammlung von Perl-Skripten, die Rasmus Lerdorf 1995 entwickelt hat, um seinen Online-Lebenslauf zu schreiben. Heute gilt dieser als Entwickler von PHP.
Da PHP serverseitig funktioniert, können wir nicht einfach eine PHP Datei öffnen, sondern wir brauchen einen Webserver. Bei uns reicht allerdings auch XAMPP um PHP zu testen, da es wesentlich einfacher zu benutzen ist und auch nichts kostet.

Aufbau Befehl (Syntax)

Wie eben erwähnt ist PHP aus Perl entstanden, entsprechend sieht auch ihre Syntax aus. Man startet ein PHP Skript immer mit einem <?php Tag und beendet ein Skript dann wieder mit folgendem Tag: ?>
Ein konkretes Beispiel zur Erstellung einer Funktion würde so aussehen: <?php
function frame() {
echo "Hello World";
}
?>
Um diese Funktion dann später zu benutzen, muss man folgende Syntax verwenden: <?php
frame();
?>

Funktionen

Hier liste ich die meiner Meinung nach wichtigsten Befehle von PHP auf.

echo $_POST["id"];
Wenn in einem HTML Formular einer Abfrage eine ID zugewiesen wird, kann auf diese Weise deren Inhalt ausgegeben werden. Das geht aber nur sofern im Formular das method="post" Attribut ist.

echo date("d.m.Y H:i:s");
Mit diesem Befehl kann das aktuelle Datum angegeben werden.


Was ich oben schon kurz angeschnitten habe, mit dem method-Attribut, will ich nochmals kurz vertiefen. Man kann bekanntlich mit method="post" Inhalte auf ein anderes File(meistens .php) "posten". Das bietet sich hauptsächlich bei Formularen an.

Die andere Möglichkeit, ist method="get" bzw. man kann dieses Attribut komplett weglassen, dann ist die Methode auch "get". Wenn man diese Methode wählt, wird der Formularinhalt der URL angehenkt, das wohl bekannteste Beispiel ist hierbei Google.

Validierung

Bevor man ein Formular mit PHP auswerten kann, müssen erstmal die Werte von einem HTML übernommen werden. Um sicherzustellen das diese auch sinngetreu sind, gibt es die Validierung. Mit der Validierung kann man Werte nach ihrem Inhalt prüfen und schauen ob sie die festgelegten Erwartungen erfüllen. Man kann dadurch invalide Werte ablehnen und darauf folgend den Benutzer dazu auffordern, die invaliden Felder nochmals auszufüllen.

Das bewerkstelligen kann man (für E-Mail) mit einem type="email" Attribut.
Auch wichtig ist das required="required" Attribut, um das Feld "obligatorisch" zu machen.

Es gibt auch noch ähnliche Möglichkeiten wie das mit der E-Mail, den meiner Meinung nach wichtigsten Bereich kann man mit folgenden Methoden abdecken:

  • a — Muss ein kleines a sein. (nicht b, nicht aa, etc.)
  • abc — Muss abc sein.
  • a* — Muss a null mal, einmal oder mehrmals sein.
  • a+ — Muss a einmal oder mehrmals sein.
  • [^a] — Muss ein Buchstabe ausser a sein.
  • a|b — Muss a oder b sein.
  • [abc] — Muss a, b oder c sein.
  • [^abc] — Muss ein Buchstabe ausser a, b und c sein.
  • [a-z] — Muss ein Buchstabe zwischen a und z sein, nur Kleinbuchstaben. (Man kann [A-Za-z] für Klein- und Grossbuchstaben benutzen und [A-Z] für nur Grossbuchstaben)
  • a.c — Muss a sein, gefolgt von einem beliebigen Buchstaben und zuletzt muss ein c sein.
  • a{5} — Muss a fünfmal sein.
  • a{5,7} — Muss a fünf- bis siebenmal sein.
  • [ -] — Muss ein Leerzeichen oder ein Bindestrich sein.
  • [0-9] — Muss eine Zahl zwischen 0 und 9 sein.
  • [Ll].*k — Muss mit einem grossen L oder kleinen l anfangen, danach müssen 0,1 oder mehrere beliebige Buchstaben sein, und zum Schluss muss ein kleines k kommen.
  • [A-Z][A-Za-z' -]+ — Muss ein einzelner Grossbuchstabe sein, gefolgt von einem oder mehreren Gross- oder Kleinbuchstaben, einem Schrägstrich, einem Apostroph oder einem Leerzeichen. Das könnte für die Validierung von Vor- und Nachnamen gut sein.
  • [0-9]{3}[ -][0-9]{3}[ -][0-9]{2}[ -][0-9]{2} — Und so einfach macht man eine Schweizer Telefonnummer. Drei Zahlen gefolgt von einem Leerzeichen oder Bindestrich, danach nochmals 3 Zahlen und ein Abstand oder ein Bindestrich und zuletzt dann noch zweimal zwei Zahlen zwischen 0 und 9, diese zwei Blöcke sind auch wieder getrennt durch ein Leerzeichen oder ein Bindestrich. Man müsste das noch ein wenig komplizierter machen, da manche Leute mit ihrer Landesvorwahl beginnen, aber dies soll nur ein Beispiel sein.
Immer das was vor dem Bindestrich ist, ist der Attributwert für das required pattern="" Attribut. Mit dieser Methode kann man nahezu jeden Fall der Validierung abdecken, es gibt nur einen kleinen Haken bei der Sache. Nämlich wenn man das Formular nicht sinngemäss ausfüllt, sprich es Validierungsfehler gibt, dann sind die Fehlermeldungen sehr unkonkret und daher nicht besonders benutzerfreundlich.

Unterschiede von PHP zu JS/jQuery

Mit beiden Skriptsprachen kann man mehr oder weniger das Gleiche machen, jedoch muss man manchmal eine Bestimmte benutzen oder manchmal bietet sich eine einfach eher an.

Die Syntax der beiden Skriptsprachen unterscheidet sich mal mehr und mal weniger.

Beim Definieren und Ausgeben einer Variable sieht es bei PHP so aus: $varName = "test";
echo $varName;

Und bei jQuery dann folgendermassen: var varName = "test";
document.write(varName);

JS erklärt

Von JavaScript(JS) hat sicher jeder schon mal gehört, jedoch sollte man JavaScript auf gar keinen Fall mit Java verwechseln, dies ist etwas komplett Anderes und darum soll es nicht gehen. Um wieder auf JavaScript zurückzukommen, JavaScript ist ebenfalls eine Skriptsprache und sie wurde ebenfalls in 1995 entwickelt. JavaScript hatte die Absicht Webseiten dynamischer zu machen, die selbe Aufgabe hat es auch heute noch. Das einzige was heute noch dazugekommen ist, ist, dass man JavaScript auch auf Servern und Microcontrollern vorfinden kann.

JavaScript wurde früher hauptsächlich clientseitig eingesetzt. JavaScript ermöglicht aber beispielsweise mit dem Framework Node.js und als JScript bzw. JScript.NET auch serverseitige Anwendungen.

Die Sprache wird auch als Skriptsprache für Spiele und Anwendungsprogramme eingesetzt. Dies ist weil der Sprachkern nur wenige Objekte enthält, deswegen kann das Skript relativ klein gehalten werden.

Warum JavaScript lernen?

Falls ihr euch fragt, warum ihr JavaScript als Web-Entwickler lernen solltet, habe ich hier eine kleine Erklärung:

  1. Man braucht HTML um den Inhalt einer Webseite zu definieren
  2. Man braucht CSS um das Aussehen einer Webseite zu bestimmen
  3. Man braucht JS um das Verhalten einer Webseite einzustellen

Aufbau Befehl (Syntax)

Die Syntax von JavaScript ähnelt jener den C-orientierten Programmiersprachen, wozu auch Java gehört. Trotz der Namensähnlichkeit und Syntax-Verwandschaft, hat JavaScript nur geringe Gemeinsamkeiten mit Java.

Verwendung

Die Verwendung von JavaScript ist sehr vielseitig, hier sind die meiner Meinung nach wichtigsten aufgelistet:

  • dynamisches Manipulieren von Webseiten(auf ziemlicher jeder (guten) Webseite)
  • Datenvalidierung(Formulare)
  • Anzeige von Dialogfenstern(Anmeldefenster)
  • Senden und Empfangen von Daten, ohne dass die Seite neu laden muss(Spiele)
  • Vorschlagen von Suchbegriffen während der Eingabe(Google)
  • Werbebanner(für Einnahmen)
  • Schreib- und Lesezugriff auf Cookies und den Web Storage innerhalb eines Browsers(personalisierte Werbung)

Praxis

Als allererstes muss man das JavaScript File korrekt ins HTML Dokument bringen. Das kann man grundsätzlich mit zwei verschiedenen Möglichkeiten machen, entweder man schreibt einfach alles innerhalb <script>
...
</script> oder man lässt dessen Inhalt leer und verweist auf ein externes JavaScript File. Dies macht man mit einem src-Attribut: <script src="scriptXY.js">
</script>
Nun kann man mit dem Skripten beginnen.
Ich liste euch hier die wichtigsten Grundlagen auf:

Variablen
Variablen wie man sie aus der Mathematik kennt gibt es auch bei JavaScript. Man verwendet sie unvorstellbar oft. Um eine zu definieren verwendet man folgende Syntax: var variableName1 = "variableZ"; oder var variableName2 = 1; oder var variableName3 = false; Beim ersten Beispiel ist die Variable ein String beziehungsweise Text, beim zweiten Beispiel ein Integer (Wert) und beim letzten Beispiel ein Boolean sprich true oder false.
Rechnen
Man kann mit normalen Zahlen sowie mit Variablen rechnen. Es gibt einerseits die üblichen Operatoren (+,-,* und /), hier sind ein paar Beispiele: var a = 5;
var b = 12;
var c = a + a + b; Die Variable c würde jetzt 22 ergeben.

Oder Division: var variable1 = 20;
var variable2 = 4;
var resultat = variable1 / variable2; Die Variable resultat ist in diesem Falle gleich 5.

Zuletzt möchte ich noch zeigen das man auch verschiedene Zeichen kombinieren kann, dabei muss man aber die Punkt-Vor-Strich Regel beachten. Wenn man dies umgehen möchte muss man mit Klammern arbeiten. var x = 2;
var y = 6;
var z = 3;
var resultat1 = x + y * z;
var resultat2 = (x + y) * z; Jeder der in der Primarschule aufgepasst hat kann jetzt sagen, weshalb sich die beiden Resultate unterscheiden. Nämlich ist resultat1, 20 und resultat2 ist gleich 24.

Aber es gibt auch weniger bekannte Zeichen wie zum Beispiel %(auch Modulo genannt). Mit % kann man den Rest einer Division ausrechnen, hier ein Beispiel dazu: var x = 9;
var y = 4;
var z = x % y; Dies würde jetzt z = 1 ausgeben weil 9 durch 4 = 2 gibt und Rest 1 ist.
Arrays
Wenn man einer Variable mehrere Werte geben will, sind Arrays die Lösung. Dadurch muss man nicht für jeden Wert eine Variable erstellen. Man kann die einzelnen Werte eines Arrays dann mit einer Index Nummer abrufen. Beispiel: Wir wollen einen Früchtekorb erstellen, darin soll es Äpfel, Bananen und Trauben haben, folglich würde die Syntax so aussehen: var Früchtekorb = ["Apfel", "Banane", "Traube"]; Jetzt kann man die einzelnen Werte abrufen, indem man die entsprechende Index Nummer anspricht. Man muss darauf achten, dass der erste Eintrag 0 ist, aber danach wird normal gezählt. Also bei unserem Beispiel wäre Apfel = 0, Banane = 1 und Traube = 2. Den Wert Banane kann man jetzt mit folgender Syntax ausgeben: Früchtekorb[1];

jQuery

jQuery ist eine kostenfreie JavaScript-Bibliothek, wessen Ziel es ist, das Navigieren und Manipulieren, mit dem D.O.M., zu vereinfachen. Ebenfalls ist jQuery die meistverwendete JavaScript-Bibliothek, ungefähr jede zweite Webseite nutzt jQuery (Stand: 3. September 2012.) und sogar 70% der 10'000 meistbesuchten Webseiten haben Anfangs 2018 jQuery im Code.
Der Erfinder von jQuery ist John Resig, dieser hat sich zwar inzwischen aus der aktiven Entwicklungsarbeit zurückgezogen und diese an das jQuery Team übergeben, aber dort arbeitet er weiterhin als Ehrenmitglied und selbstverständlich kann er auch als Entwickler und somit Fachexperte aushelfen.

Wenn man jQuery nutzen will, sollte man immer das aktuellste CDN verwenden, da diese auch eher für neue Browser optimiert sind, man findet die jQuery-Libraries hier.

Wenn man mit jQuery durchstarten will, sollte man definitiv diese Webseite anschauen.
